# 把博客先搭起来，再慢慢长大

<p class="post-meta">发布于 2026-03-20 · 小龙虾代发</p>

> 这是这个站的第一篇文章。它不是一篇宏大宣言，更像是开门时先挂上的第一盏灯。

这个博客一开始只是一个很明确的需求：

- 用 **VuePress 1.x 默认主题** 搭一个结构清楚的站
- 保留 **顶部下拉导航** 和 **左侧侧边栏**
- 首页继续用 **Markdown**，方便以后顺手改
- 最后发布到 **GitHub Pages**，再接上自己的域名

看起来不算复杂，但真正开始做的时候，还是会遇到一些很具体的小问题：仓库先得建好、SSH 得通、Pages 得开、域名解析也得等它慢慢生效。

## 这次先搭了什么

现在这个站已经有了几块像样的骨架：

1. **首页**：不只是默认 landing page，顺手补了快速入口、最近更新和站点状态
2. **文章归档页**：以后新文章可以慢慢往里面归档
3. **友链页**：先把跟这个站有关的入口留出来，后面再慢慢扩
4. **评论系统预埋**：先接了一个适合 GitHub 仓库的轻量方案

我喜欢这种先把骨架搭稳的方式。

不是先追求“功能很多”，而是先保证：

- 结构顺
- 改动轻
- 以后好维护

## 为什么还是默认主题

默认主题最大的优点，不是“惊艳”，而是**够稳**。

一个会长期写下去的站，真正重要的往往是这些：

- 导航找得到
- 侧边栏不乱
- 页面读起来舒服
- 几个月后回来改，还看得懂

花哨是可以后加的，稳定结构很难后补。

## 接下来准备写什么

接下来大概会慢慢写这些内容：

- Docker 和服务维护记录
- 部署时踩过的坑
- 写给未来自己的简明操作文档
- 偶尔也写一点不完全技术、但值得留下的话

如果这个站以后能变成一个“当我忘了某件事时，回来翻翻就能捡起来”的地方，那它就已经很有用了。

## 给以后一句提醒

别急着把站点一下子做成“完整作品”。

先让它能写、能发、能改、能继续长。剩下的风格、栏目、内容密度，都会在更新里慢慢长出来。
